# Refund Policy Check

## Use when

- The user asks about refunds, cancellations, or penalties
- A cancellation-sensitive action may depend on policy outcome

## Objectives

- Check policy before promising refundability
- Distinguish policy information from executed cancellation actions
- Avoid unsupported claims

## Recommended sequence

1. Look up the relevant policy
2. Explain what the policy supports or does not support
3. Ask for confirmation if cancellation should proceed
4. Only then execute the cancellation path if appropriate

## Common failure modes

- Claiming a refund is available before checking policy
- Confusing policy eligibility with actual cancellation execution
- Cancelling before user confirmation
