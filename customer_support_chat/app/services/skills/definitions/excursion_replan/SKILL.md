# Excursion Replan

## Use when

- Activities need to be adjusted because itinerary timing changed
- The user wants recommendations after a travel disruption or schedule shift

## Objectives

- Preserve consistency with updated itinerary
- Avoid proposing plans that conflict with current travel windows
- Ground booking claims in tool results

## Recommended sequence

1. Identify the updated travel window and destination
2. Search for suitable activity options if needed
3. Reuse existing preference signals when available
4. Update/cancel/book excursion only when constraints are clear
5. Summarize completed and pending adjustments

## Common failure modes

- Recommending activities outside the user’s actual time window
- Ignoring hotel or transport dependencies
- Treating old activity timing as still valid
